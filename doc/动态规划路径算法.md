## 动态规划路径算法

编写算法函数，根据tree_data结构，推演出选中到指定N个载具最终所需RP最少的路径。

```js
function findShortestPathToVehicle() {
  const treeDataStore = useTreeDataStore();
  const { tree_data, researchable_set, selected_state_map, total_stats } = storeToRefs(treeDataStore);
}
```



### 基本信息

在tree_data中不断推演，直到规划出选中到指定N个载具最终所需RP最少的路径，最终的选中态直接覆盖到selected_state_map。

每个rank对象具备一个unlock_quantity字段，其表示解锁下一级researchable_vehicles节点所需的当前等级节点选中数（包括researchable_vehicles和premium_vehicles）。

约定：仅researchable_vehicles中的节点可以作为最终目标节点，premium_vehicles中的节点通常仅作为凑数用的可选节点。

tree_data结构剖解：第一层包含N个对象，每个对象表示一个等级区域：

- researchable_vehicles为银币载具，也就是需要计算RP/SP进行动态规划的区域
- premium_vehicles为高级载具，其下节点仅作为凑数使用

```js
{
  "rank": "V",
  "researchable_vehicles": [
    [],
    [],
    [],
  ],
  "premium_vehicles": [
    [],
    []
  ],
  "unlock_quantity": 0
}
```

researchable_vehicles和premium_vehicles都是数组，其下第一级表示列：

```js
{
  "rank": "V",
  "researchable_vehicles": [
    [], // 5级银币载具区域的第一列
    [], // 5级银币载具区域的第二列
    [], // 5级银币载具区域的第三列
  ],
  "premium_vehicles": [
    [], // 5级高级载具区域的第一列
    []  // 5级高级载具区域的第二列
  ],
  "unlock_quantity": 0
}
```

每个等级区域对象中的“列”最终渲染在UI层面就是同一列，例如：

- 在UI层面，它们本质上就是同一列，只是不同等级而已，你需要清楚这个概念。

```js
{
  "rank": "V",
  "researchable_vehicles": [
    [], // 5级银币载具区域的第一列
  ],
}
{
  "rank": "VI",
  "researchable_vehicles": [
    [], // 6级银币载具区域的第一列
  ],
}
```

假设用户A传入的result_targets为["s_166"]，那么你需要从tree_data的最上面开始逐级选中足够数量的researchable_vehicles节点，一直选到"s_166"这个节点，但这条路线必需符合用户指定的priority_column，也就是说，除了主线列意外，凑数时优先按照用户指定的列来进行（即使这不一定是总RP最少的路径），如果用户没有指定priority_column，那么你将要按照最短路径来规划（也就是总RP最少的方案）。在规划的过程中，你会遇到type: "multiple"的节点，这是一个组节点，通常组节点下的子节点所需RP更少，因此在凑数时可以优先考虑选中这些子节点来凑数。



### 工作前准备：

读取store中的selected_state_map，将当前选中的researchable_vehicles导出至一个数组中，这个数组下的每个data_unit_id对应动态规划的最终目标节点。

```js
const result_targets = [data_unit_id, data_unit_id, data_unit_id...]
```

**函数接收参数：**

targets：指定N个最终的目标节点。

```js
// 传入示例：
const targets = [
  { data_unit_id: "mk9c_iaf", rank: "V" },
  { data_unit_id: "s_199", rank: "VI" }
]
function findShortestPathToVehicle({ targets })
```

planned_prems：指定N个预选的premium_vehicles节点，这些预选节点在规划过程中是非常重要的额外凑数节点。

```js
// 传入示例
const planned_prems = [
  { data_unit_id: "mk9e_weisman", rank: "V" }
]
function findShortestPathToVehicle({ targets, planned_prems })
```

priority_column：指定N个researchable_vehicles下的优先凑数列，具备优先级特性。

- 传入示例 [0, 2]：表示优先以第一列凑数，其次以第三列凑数

```js
function findShortestPathToVehicle({ target, planned_prems, priority_column })
```

### unlock_quantity字典值

不要直接读取每个等级区域对象下的unlock_quantity字段，因为它们全部都是0，通过访问字典值来获取实际对应的unlock_quantity。

```js
import { unlock_quantitys } from "@/utils/dict";

unlock_quantitys[types.value.country_code][types.value.vehicle_type][等级，如"V"]
```
